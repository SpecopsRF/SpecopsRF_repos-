**ТЗ №1**
- Сделать гит репозиторий
- Добывить ssh ключи в профиле git
- Создать тестовый проект, который отвечает следующим требованиям:
    - Это простое веб приложение
    - У него есть простая база данных (можно для простоты использовать psql или sqlite)
    - Оно собирается в докере
    - У него есть секреты, которые берутся из файла с секретами (например пароль от бд)
    - Есть dockerfile для сборки образа веб приложения
    - Есть docker-compose.yaml для автоматический сборки и запуска всего веб приложения
- Залить в репозиторий тестовый код, добавив перед этим в .gitignor файл с секретами

**ТЗ №2**
- Сделать ранер для автоматической сборки docker образа веб приложения
- Сделать в git docker regystry, куда будет помещаться собранный докер образ
- Добавить все секреты в git переменные, чтобы при сборке они брались из git


**ТЗ №3**

- После этого нужно научить git разворачивать созданное веб приложение на vps

**ТЗ №4**
- Добавить сканирование sast кода на этапе сборки (нужно встроить в пайплайн), используем все тот же бандит
